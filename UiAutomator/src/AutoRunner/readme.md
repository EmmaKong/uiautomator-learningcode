
利用Android自动化测试工具UiAutomator开发“模拟用户真实使用场景的自动化压力测试工具”

开发目的：模拟用户的真实使用场景，挖掘用户使用过程中可能碰到的系统性能问题，如：卡顿，死机，闪退等问题。

涉及的相关应用：
后台任务包括：下载任务，播放音乐，QQ，微信，微博等聊天工具，文件管理的文件传输，拷贝粘贴等
前台任务包括：视频播放，浏览器，相册浏览，相机拍照，摄像等
其他条件：各类传感器、移动网络、wifi、蓝牙、GPS开关切换等
SD卡操作，填充data空间，用RAM等极限操作
资源抢占，CPU极限操作，GPU极限操作，内存极限操作。

我的工作：模拟各种前后台任务的常用基本操作，实现不同任务之间的频繁切换


涉及到的app，及相应的操作：
应用商店：启动，搜索并下载或打开一个app，更新所有的可更新app，退出并在后台完成下载；

文件管理：启动，删除所有安装包，实现图片文件的拷贝和粘贴，退出并在后台完成文件传输；

i音乐：启动，搜索并下载音乐，搜索并播放音乐，随机播放音乐，退出并在后台播放音乐；

新浪微博：启动，刷新，浏览主页，浏览好友圈，浏览我的微博，发微博，清除缓存，退出至后台运行；

QQ：启动，刷新信息，查看未阅读信息，浏览QQ空间，账户设置（更换在线状态），添加好友，退出至后台运行；

微信：启动，查看未阅读信息，浏览我的相册，浏览朋友圈，发朋友圈，阅读订阅号新消息，退出至后台运行；

相机：启动，选择特定模式拍照，录制视频，退出；

相册：启动，浏览全部照片，退出；

手机百度：启动，搜索并查看内容，更换皮肤，退出；

爱奇艺：启动，搜索并下载视频，搜索并在线观看视频，观看已缓存视频；
